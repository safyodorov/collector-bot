  <action>
    Create `src/keyboards/navigation.ts` importing `InlineKeyboard` from `grammy` and `CATEGORY_MAP`, `CATEGORY_TAGS` from `../config.js`.

    **1. buildCategoryKeyboard(): InlineKeyboard** (per NAVG-01)
    - Iterate `Object.entries(CATEGORY_MAP)` in insertion order
    - Each button: `kb.text(cat.label, 'nav:' + key)`
    - 2 buttons per row: call `kb.row()` after every 2nd button
    - If odd number of buttons, add `kb.row()` after last
    - Final row: `kb.text('Отмена', 'nav:cancel')`
    - callback_data format: `nav:{category_key}` (e.g. `nav:бизнес`, `nav:inbox`)
    - Expected layout:
      ```
      [ Бизнес    ] [ Ландшафт ]
      [ ТОС       ] [ Рецепты  ]
      [ Семья     ] [ Новости  ]
      [ Идеи      ] [ Inbox    ]
      [        Отмена          ]
      ```

    **2. buildSubcategoryKeyboard(categoryKey: string): InlineKeyboard** (per NAVG-02)
    - Lookup `CATEGORY_MAP[categoryKey]` — throw if not found or no subs
    - Iterate subcategories, 2 per row
    - callback_data: `nav:{categoryKey}_{subKey}` (e.g. `nav:бизнес_wb`)
    - After subcategories: `kb.text('← Просто в ' + cat.label, 'nav:' + categoryKey + '_root').row()`
    - Final row: `kb.text('Отмена', 'nav:cancel')`
    - Example for бизнес:
      ```
      [ WB          ] [ Ozon       ]
      [ Поставщики  ] [ Финансы    ]
      [ Аналитика   ] [ Контент    ]
      [ Налоги      ]
      [ ← Просто в Бизнес        ]
      [        Отмена             ]
      ```

    **3. buildTagKeyboard(categoryKey: string, selectedTags: string[]): InlineKeyboard** (per TAGS-01, TAGS-02, TAGS-03)
    - Lookup `CATEGORY_TAGS[categoryKey]` — if not found, use empty array
    - For each tag: if `selectedTags.includes(tag)` prefix with "✅ ", callback_data `tag:${tag}`
    - 2 per row
    - After tags: `kb.text('Готово (' + selectedTags.length + ')', 'tag:done').row()` (shows count if any selected)
    - If `selectedTags.length === 0`: button text is just "Готово" not "Готово (0)"
    - Then: `kb.text('Без тегов', 'tag:none').row()` (per TAGS-02)
    - Then: `kb.text('✏️ Написать свой', 'tag:custom').row()` (per TAGS-03)
    - Then: `kb.text('Отмена', 'nav:cancel')`

    **4. buildTitleKeyboard(): InlineKeyboard**
    - Single button: `kb.text('Пропустить', 'title:skip')`
    - Same as current hashtags.ts

    **5. buildDuplicateKeyboard(): InlineKeyboard**
    - `kb.text('Сохранить как новую', 'dup:new').text('Отменить', 'dup:cancel')`
    - Same as current hashtags.ts

    Do NOT modify or delete `src/keyboards/hashtags.ts` — it's still imported by the current bot.ts. Plan 03 will switch the import to navigation.ts.

    callback_data byte safety: all callback_data values use short keys. The longest possible is `nav:ландшафт_благоустройство` = ~51 bytes UTF-8, well under Telegram's 64-byte limit.

  <action>
Create `scripts/test-webdav.ts` -- a standalone script that runs against real Yandex.Disk to verify all WebDAV operations work. Run with `npx tsx scripts/test-webdav.ts`.

The script must test these operations in order:

1. **Auth check**: Call `initWebDAV()`. If it throws, log error and exit 1.

2. **Create vault structure**: Call `initVault()`. Log success.

3. **Idempotency check**: Call `initVault()` again. Must not throw (405 on existing dirs is OK).

4. **Upload text file with Cyrillic name**: `putFile(\`\${VAULT_PATH}/Inbox/тест_\${Date.now()}.md\`, 'Привет, мир! Тестовая заметка.', 'text/markdown; charset=utf-8')`. Log the path.

5. **Check file exists**: `exists(\`\${VAULT_PATH}/Inbox/тест_\${Date.now()}.md\`)` should return true. Also check a non-existent path returns false.

6. **Read file back**: `getFile()` on the uploaded text file. Verify content equals `'Привет, мир! Тестовая заметка.'`.

7. **Upload binary file**: Create a small test buffer (`Buffer.from([0xFF, 0xD8, 0xFF, 0xE0, 0x00, 0x10, 0x4A, 0x46, 0x49, 0x46])` -- JFIF header bytes). `putFile(\`\${VAULT_PATH}/attachments/test_\${Date.now()}.jpg\`, buffer, 'image/jpeg')`.

8. **Verify binary exists**: `exists()` on the uploaded binary file.

Each test step should:
- Log `[TEST] Step N: description...` before running
- Log `[PASS] description` on success
- Log `[FAIL] description: error` on failure and set exitCode = 1
- Continue to next test (don't stop on first failure)

At the end, log summary: `[RESULT] N/M tests passed` and `process.exit(exitCode)`.

Store the generated filenames in variables so exists/getFile checks use the same path as the upload.

Imports:
```typescript

```

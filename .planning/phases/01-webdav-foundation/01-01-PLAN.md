  <action>
Create `src/services/webdav.ts` as a focused module wrapping native `fetch()`. No npm dependencies. No classes. Module-level `dirCache = new Set<string>()` for MKCOL caching.

The module must export these functions:

1. **`encodePath(path: string): string`** -- Split on `/`, `encodeURIComponent()` each non-empty segment, rejoin with `/`. This handles Cyrillic folder/file names.

2. **`WebDAVError` class** -- extends Error with `status: number`, `method: string`, `path: string` fields. Constructor: `super(\`WebDAV ${method} ${path}: ${status} ${message}\`)`, set `this.name = 'WebDAVError'`.

3. **`withRetry<T>(operation: () => Promise<T>, maxRetries = 3, baseDelay = 1000): Promise<T>`** -- Generic retry wrapper. For each attempt:
   - Call operation
   - If result is a Response with status 429 or 500-504: wait `baseDelay * 2^(attempt-1)` ms, retry
   - If network error (TypeError from fetch): wait and retry
   - On final attempt: return the response as-is (let caller handle status)
   - Log retries: `console.log(\`[WEBDAV] Retry \${attempt}/\${maxRetries} after \${delay}ms\`)`

4. **`putFile(path: string, content: Buffer | string, contentType = 'text/markdown; charset=utf-8'): Promise<void>`** -- PUT to `${WEBDAV_URL}${encodePath(path)}` with `Authorization: OAuth ${YANDEX_DISK_TOKEN}` and `Content-Type` header. Accept 201 (created) and 204 (overwritten). On 202 (async processing): poll with `exists()` every 1s, up to 30s, then throw if still not found. On other non-success: throw WebDAVError. Use withRetry wrapper.

5. **`ensureDir(path: string): Promise<void>`** -- Split path into segments, build each level. For each level: skip if in `dirCache`. MKCOL with auth header. Accept 201 (created) and 405 (already exists). On 409 (parent missing): should not happen since we create sequentially, but throw WebDAVError. Add to `dirCache` on success.

6. **`exists(path: string): Promise<boolean>`** -- PROPFIND with `Depth: 0` header. 207 = true, 404 = false. Other status = throw WebDAVError. Use withRetry.

7. **`getFile(path: string): Promise<string | null>`** -- GET with auth header. 200 = return `res.text()`, 404 = return null. Other = throw WebDAVError. Use withRetry.

8. **`initWebDAV(): Promise<void>`** -- Auth verification. PROPFIND on `/` with Depth:0. If 401: throw `new Error('WebDAV auth failed: invalid or expired YANDEX_DISK_TOKEN')`. If 207: `console.log('[WEBDAV] Authentication verified')`. Other: throw with status.

Import from config.ts using relative path: `import { YANDEX_DISK_TOKEN, WEBDAV_URL } from '../config.js'`

Note the `.js` extension in imports -- project uses ESM with NodeNext module resolution.

Internal helper `authHeaders()` returns `{ 'Authorization': \`OAuth \${YANDEX_DISK_TOKEN}\` }`.

Target size: ~120-150 lines. No XML parsing. No external dependencies.
